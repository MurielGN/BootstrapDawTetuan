Range



Range Steps

Lo que hacemos con el range es crear una caja con una cifra a la derecha, una linea de movimiento la cual dependiendo de donde la colocaremos saldra una cifra u otra, dependiendo de donde queramos llegar.
Esto sucede ya que al crear la etiqueta input, ponemos tipo range y pondremos un minimo y un maximo, y despues con un script crearemos dos constantes para poder hacer el efecto y que funcione, contandolos de uno en uno.

<h2>¿La variedad de productos presentada en el menú es suficiente?</h2>
<div class="box">
<div class="slider">
<input type="range" min="0" max="10" value="10">
</div>
<div class="valuerange">0</div>
</div>
<script>
const slider = document.querySelector("#range");
const value = document.querySelector(".valuerange");
value.textContent = slider.value;
slider.oninput = function(){
value.textContent = this.value;
}
</script>

¿La variedad de productos presentada en el menú es suficiente?


0

Input Groups



Input Group & Input Group Size

La clase .input-group es un contenedor para mejorar una entrada agregando un icono, texto o un botón delante o detrás del campo de entrada como "texto de ayuda" Se puede usar la clase .input-group-sm para input-groups pequeños y .input-group-lg para input-groups grandes:

<h4>¿Cuánto estaría dispuesto a pagar por nuestro menú diario?</h4>
<!--Diseñamos un div con input-group de tamaño grande ("lg")-->
<div class="input-group mb-3 input-group-lg">
<input type="text" class="form-control" placeholder="Introduzca un importe" name="euros">
<!--Con la etiqueta span agregamos un botón tras el input, en este caso con el símbolo del €-->
<span class="input-group-text">€</span>
</div>

¿Cuánto estaría dispuesto a pagar por nuestro menú diario?

€

Form Validation



Validation

Puede usar diferentes clases de validación para proporcionar comentarios valiosos a los usuarios. Agregue .was-validated o .needs-validation al elemento form, dependiendo de si desea proporcionar comentarios de validación antes o después de enviar el formulario. Los campos de entrada tendrán un borde verde (válido) o rojo (no válido) para indicar lo que falta en el formulario. También puede agregar un mensaje .valid-feedback o .invalid-feedback para decirle al usuario explícitamente qué falta o qué debe hacerse antes de enviar el formulario.

<form action="/action_page.php" class="was-validated">
<div class="mb-3 mt-3">
<label for="uname" class="form-label">Usuario:</label>
<input type="text" class="form-control" id="uname" placeholder="Escribe el usuario" name="uname" required>
<div class="valid-feedback">Correcto.</div>
<div class="invalid-feedback">Por favor introduce un nombre.</div>
</div>
<div class="mb-3">
<label for="pwd" class="form-label">Contraseña:</label>
<input type="password" class="form-control" id="pwd" placeholder="Escribe la contraseña" name="pswd" required>
<div class="valid-feedback">Valida.</div>
<div class="invalid-feedback">Por favor introduce una contraseña.</div>
</div>
<div class="form-check mb-3">
<input class="form-check-input" type="checkbox" id="myCheck" name="remember" required>
<label class="form-check-label" for="myCheck">Estoy de acuerdo con...</label>
<div class="valid-feedback">Perfecto.</div>
<div class="invalid-feedback">Acepata los terminos para continuar.</div>
</div>
<button type="submit" class="btn btn-primary">Enviar</button>
</form>
